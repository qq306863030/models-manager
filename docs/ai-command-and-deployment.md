# `ai` 命令使用 & 部署流程

## 一、`ai` 命令简介

`ai` 是一个本地 CLI 工具，通过本地启动代理服务 + 远程 SSH 的方式，实现对远程服务器的命令执行和文件操作。

### 基本用法

```bash
ai "<自然语言指令>"
```

将意图用自然语言描述，`ai` 会自动解析并执行。示例：

```bash
# 查询远程 Docker 容器
ai "帮我获取远程服务器中运行中的docker容器的实例"

# 读取远程日志
ai "查看远程服务器上指定日志文件中包含某关键词的行"

# 远程执行命令
ai "帮我执行：docker exec 容器名 命令"

# 上传文件
ai "帮我上传本地某个目录下的所有文件到远程服务器的指定目录"
```

> 远程文件路径建议使用 **绝对路径**，避免因默认工作目录导致找不到文件。

## 二、部署流程

### 2.1 本地构建

```bash
cd 项目目录
npm run build
```

构建产物输出到 `dist/` 目录，包括：
- 后端编译的 JS 文件
- 前端打包的 `public/` 静态资源

### 2.2 上传到远程服务器

使用 `ai` 命令一键部署：

```bash
ai "帮我部署：1. 删除远程服务器上'/roman/models-manager/manager/'中dist目录的所有内容 2. 把我本地的dist目录下的所有文件上传到远程服务器的对应目录 3. 重启docker容器'ai-manager'"
```

`ai` 会自动完成三步：
1. **删除**远程旧 dist
2. **上传**本地新 dist
3. **重启** Docker 容器
