# MCP 记录功能实施计划

## 概述

在模型管理页面添加「MCP记录」按钮，点击后打开 JSON 编辑器弹窗，支持编辑、验证、格式化 JSON 配置。后端新增数据库表存储，按用户隔离。

---

## 改动 1：后端 — 新增 MCP 记录表

**文件：`src/config/database.ts`**

新增 `mcp_records` 表：

```sql
CREATE TABLE IF NOT EXISTS mcp_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  content TEXT NOT NULL DEFAULT '{}',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
)
```

并导出操作函数：
- `getMcpRecord(userId)` — 查询
- `upsertMcpRecord(userId, content)` — 插入或更新

---

## 改动 2：后端 — 新增 MCP 路由

**新建文件：`src/routes/mcpRecords.ts`**

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/mcp-records` | 获取当前用户的 MCP 记录 |
| `POST` | `/api/mcp-records` | 保存/更新当前用户的 MCP 记录 |

- 通过 `req.headers['x-username']` 获取用户名，转为 `userId`
- POST 时验证 `content` 是否为合法 JSON 字符串
- 返回 `{ success: true/false, data/content, message }`

**修改文件：`src/app.ts`** — 注册新路由：
```typescript
import mcpRecordsRouter from './routes/mcpRecords';
app.use('/api/mcp-records', mcpRecordsRouter);
```

## 改动 4：前端 — 新增 API 服务

**新建文件：`frontend/src/api/mcpRecordService.ts`**

```typescript
import request from './models';

export const getMcpRecord = () => 
  request.get<{ success: boolean; data: { content: string } | null }>('/mcp-records');

export const saveMcpRecord = (content: string) => 
  request.post<{ success: boolean; message: string }>('/mcp-records', { content });
```

## 改动 5：前端 — 新增 MCP 记录弹窗组件

**新建文件：`frontend/src/components/McpRecordDialog/index.vue`**

### 结构

```
<el-dialog title="MCP 记录" width="800px">
  <div class="editor-container" ref="editorRef">
    <!-- CodeMirror 挂载点 -->
  </div>
  
  <template #footer>
    <el-button @click="handleCancel">取消</el-button>
    <el-button type="primary" @click="handleSubmit" :loading="loading">提交</el-button>
  </template>
</el-dialog>
```

### 功能

1. **打开时**：调用 `GET /api/mcp-records` 加载已有内容，注入编辑器
2. **编辑器能力**：
   - JSON 语法高亮
   - 行号显示
   - 括号匹配
   - 自动缩进
3. **提交时**：
   - 用 `JSON.parse()` 验证合法性
   - 合法 → `JSON.stringify(parsed, null, 2)` 格式化回写编辑器
   - 调用 `POST /api/mcp-records` 保存
   - 不合法 → `ElMessage.error` 提示具体错误位置
4. **取消时**：关闭弹窗（如有未保存修改提示？先保持简单，不提示）

### 编辑器初始化

使用 `onMounted` + `nextTick` 确保 DOM 渲染后挂载 CodeMirror：

```typescript
const editorRef = ref<HTMLDivElement>()
let editorView: EditorView | null = null

const initEditor = (content: string) => {
  if (editorView) { editorView.destroy(); editorView = null }
  
  const state = EditorState.create({
    doc: content || '{}',
    extensions: [
      basicSetup,
      json(),
      oneDark,
      EditorView.updateListener.of((update) => {
        // 可选：实时跟踪修改状态
      }),
    ],
  })
  editorView = new EditorView({ state, parent: editorRef.value! })
}
```

### 保持一致的弹窗模式

遵循 SettingsDialog 的 Pattern B（独立组件 → `defineExpose({ openDialog })` → 父组件 ref 调用）。
